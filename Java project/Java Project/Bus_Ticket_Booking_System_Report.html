ABSTRACT

BUS TICKET BOOKING SYSTEM

The Bus Ticket Booking System is a Java-based application developed using J2SE with integrated database support to automate the management of bus ticket reservations. The system is designed to perform essential operations such as booking tickets, viewing seat availability, canceling bookings, and searching reservations through a menu-driven console interface. By utilizing JDBC for database connectivity with MySQL, the application ensures persistent data storage, improved data accuracy, and efficient retrieval of booking details, thereby reducing manual effort and errors.

The system follows object-oriented programming principles and adopts a modular architecture to enhance maintainability and scalability. Key features include route management, seat allocation with visual seat maps, double-booking prevention using database transactions, input validation, and exception handling to ensure reliable execution. The application implements a layered architecture with separate Model, DAO (Data Access Object), Service, and Presentation layers.

1. INTRODUCTION

In today's fast-paced world, efficient transportation management is crucial for both travelers and service providers. The Bus Ticket Booking System addresses the need for a streamlined, automated approach to managing bus reservations. This project implements a complete ticket booking solution using Java programming language with MySQL database integration.

Traditional manual ticket booking systems are prone to errors, time-consuming, and difficult to manage. Issues such as double booking, lost records, and inefficient seat allocation are common. This computerized system eliminates these problems by providing a reliable, fast, and user-friendly interface for managing bus ticket reservations.

1.1 Problem Statement

Manual bus ticket booking systems face several challenges including:

  • Difficulty in tracking available seats across multiple buses
  • Risk of double-booking the same seat
  • Time-consuming search for booking records
  • Lack of centralized data management
  • Inefficient cancellation and refund processing

1.2 Proposed Solution

The Bus Ticket Booking System provides a comprehensive solution with features including:

  • Centralized database for all booking information
  • Real-time seat availability tracking
  • Transaction-based booking to prevent double-booking
  • Visual seat map for easy selection
  • Quick search and retrieval of booking records
  • Simple cancellation process

1.3 Scope of the Project

This project covers the core functionalities required for a bus ticket booking system. It includes route management, bus management, seat allocation, booking creation, cancellation, and booking search features. The system is designed as a console-based application suitable for learning purposes and can be extended to a web-based system in the future.

4. SYSTEM ARCHITECTURE

The Bus Ticket Booking System follows a 3-tier layered architecture that separates concerns and promotes maintainability. The architecture consists of the following layers:

4.1 Architecture Diagram

+----------------------------------------------------------+ | PRESENTATION LAYER | | (BusTicketApp.java - Console UI) | | - Main Menu Display | | - User Input Handling | | - Output Formatting (ConsoleUI.java) | +----------------------------+-----------------------------+ | v +----------------------------------------------------------+ | BUSINESS LOGIC LAYER | | (BookingService, RouteService, SeatService, etc.) | | - Input Validation | | - Business Rules | | - Transaction Management | +----------------------------+-----------------------------+ | v +----------------------------------------------------------+ | DATA ACCESS LAYER (DAO) | | (RouteDAO, BusDAO, SeatDAO, BookingDAO) | | - CRUD Operations | | - SQL Query Execution | | - Result Set Mapping | +----------------------------+-----------------------------+ | v +----------------------------------------------------------+ | DATABASE LAYER | | (MySQL Database) | | - routes, buses, seats, bookings tables | +----------------------------------------------------------+

4.2 Layer Descriptions

Presentation Layer

This layer handles all user interactions through the console interface. It displays menus, receives user input, and presents formatted output. The ConsoleUI utility class provides methods for creating visually appealing console output with colors and table formatting.

Business Logic Layer (Service Layer)

Contains the core business logic of the application. This layer validates inputs, enforces business rules (like preventing past-date bookings), and coordinates between the presentation and data access layers. It manages database transactions for operations requiring atomicity.

Data Access Layer (DAO Layer)

Responsible for all database interactions. Each entity (Route, Bus, Seat, Booking) has its own DAO class that handles CRUD (Create, Read, Update, Delete) operations. This layer abstracts the database implementation from the business logic layer.

Database Layer

The MySQL database stores all persistent data. It includes tables for routes, buses, seats, and bookings with proper foreign key relationships to maintain data integrity.

8. DOUBLE BOOKING PREVENTION

One of the critical features of the Bus Ticket Booking System is preventing double booking of the same seat. This is achieved using database transactions and row-level locking.

8.1 The Problem

In a multi-user environment, two users might try to book the same seat simultaneously. Without proper handling, both bookings could succeed, resulting in a conflict.

8.2 The Solution

The system uses the following approach to prevent double booking:

  1. Start Transaction: Begin a database transaction with auto-commit disabled.
  2. Lock the Row: Use SELECT ... FOR UPDATE to lock the seat row.
  3. Check Availability: Verify the seat is still available.
  4. Insert Booking: If available, insert the booking record.
  5. Commit or Rollback: Commit if successful, rollback if not.

8.3 Implementation Code

// In BookingService.java public boolean bookSeat(int busId, String seatNumber, ...) { Connection conn = null; try { // Get new connection and start transaction conn = DatabaseConnection.getInstance().getNewConnection(); conn.setAutoCommit(false); // Check availability with row lock if (!seatDAO.isSeatAvailable(conn, seatId, travelDate)) { conn.rollback(); ConsoleUI.printError("Seat already booked!"); return false; } // Create booking int bookingId = bookingDAO.createBooking(conn, booking); // Commit transaction conn.commit(); ConsoleUI.printSuccess("Booking successful!"); return true; } catch (SQLException e) { if (conn != null) conn.rollback(); return false; } }

8.4 SQL Query for Row Locking

-- Check seat availability with row lock SELECT COUNT(*) as cnt FROM bookings WHERE seat_id = ? AND travel_date = ? AND status = 'CONFIRMED' FOR UPDATE;

The FOR UPDATE clause locks the matched rows, preventing other transactions from modifying them until the current transaction is complete.

9. SQL QUERIES

9.1 Table Creation Queries

-- Create Routes Table CREATE TABLE routes ( route_id INT PRIMARY KEY AUTO_INCREMENT, source VARCHAR(100) NOT NULL, destination VARCHAR(100) NOT NULL, fare DECIMAL(10, 2) NOT NULL, duration_minutes INT NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); -- Create Buses Table CREATE TABLE buses ( bus_id INT PRIMARY KEY AUTO_INCREMENT, route_id INT NOT NULL, bus_number VARCHAR(20) NOT NULL UNIQUE, bus_type ENUM('AC', 'NON_AC', 'SLEEPER', 'SEMI_SLEEPER') NOT NULL, total_seats INT NOT NULL DEFAULT 40, departure_time TIME NOT NULL, FOREIGN KEY (route_id) REFERENCES routes(route_id) ); -- Create Seats Table CREATE TABLE seats ( seat_id INT PRIMARY KEY AUTO_INCREMENT, bus_id INT NOT NULL, seat_number VARCHAR(5) NOT NULL, seat_type ENUM('WINDOW', 'AISLE', 'MIDDLE') NOT NULL, FOREIGN KEY (bus_id) REFERENCES buses(bus_id), UNIQUE KEY uk_bus_seat (bus_id, seat_number) ); -- Create Bookings Table CREATE TABLE bookings ( booking_id INT PRIMARY KEY AUTO_INCREMENT, seat_id INT NOT NULL, passenger_name VARCHAR(100) NOT NULL, passenger_phone VARCHAR(15) NOT NULL, passenger_email VARCHAR(100), travel_date DATE NOT NULL, booking_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP, status ENUM('CONFIRMED', 'CANCELLED') DEFAULT 'CONFIRMED', FOREIGN KEY (seat_id) REFERENCES seats(seat_id) );

9.2 Data Retrieval Queries

-- Get all routes SELECT route_id, source, destination, fare, duration_minutes FROM routes ORDER BY source, destination; -- Get buses for a route with available seats SELECT b.*, r.source, r.destination, r.fare, (b.total_seats - COUNT(bk.booking_id)) as available FROM buses b JOIN routes r ON b.route_id = r.route_id LEFT JOIN seats s ON b.bus_id = s.bus_id LEFT JOIN bookings bk ON s.seat_id = bk.seat_id AND bk.travel_date = ? AND bk.status = 'CONFIRMED' WHERE b.route_id = ? GROUP BY b.bus_id; -- Get available seats for a bus on a date SELECT s.*, CASE WHEN b.booking_id IS NOT NULL THEN TRUE ELSE FALSE END as is_booked FROM seats s LEFT JOIN bookings b ON s.seat_id = b.seat_id AND b.travel_date = ? AND b.status = 'CONFIRMED' WHERE s.bus_id = ? ORDER BY s.seat_number;

9.3 Data Modification Queries

-- Insert new booking INSERT INTO bookings (seat_id, passenger_name, passenger_phone, passenger_email, travel_date, status) VALUES (?, ?, ?, ?, ?, 'CONFIRMED'); -- Cancel booking UPDATE bookings SET status = 'CANCELLED' WHERE booking_id = ? AND status = 'CONFIRMED'; -- Get booking details by ID SELECT b.*, s.seat_number, s.seat_type, bus.bus_number, bus.bus_type, bus.departure_time, r.source, r.destination, r.fare FROM bookings b JOIN seats s ON b.seat_id = s.seat_id JOIN buses bus ON s.bus_id = bus.bus_id JOIN routes r ON bus.route_id = r.route_id WHERE b.booking_id = ?; -- Search bookings by phone number SELECT b.*, s.seat_number, r.source, r.destination FROM bookings b JOIN seats s ON b.seat_id = s.seat_id JOIN buses bus ON s.bus_id = bus.bus_id JOIN routes r ON bus.route_id = r.route_id WHERE b.passenger_phone LIKE ? ORDER BY b.travel_date DESC;

9.4 Aggregate Queries

-- Get available seat count for a bus on a date SELECT COUNT(*) as available FROM seats s WHERE s.bus_id = ? AND s.seat_id NOT IN ( SELECT seat_id FROM bookings WHERE travel_date = ? AND status = 'CONFIRMED' ); -- Get booking statistics SELECT COUNT(*) as total_bookings, SUM(CASE WHEN status = 'CONFIRMED' THEN 1 ELSE 0 END) as confirmed, SUM(CASE WHEN status = 'CANCELLED' THEN 1 ELSE 0 END) as cancelled FROM bookings; -- Get popular routes SELECT r.source, r.destination, COUNT(b.booking_id) as total_bookings FROM routes r JOIN buses bus ON r.route_id = bus.route_id JOIN seats s ON bus.bus_id = s.bus_id JOIN bookings b ON s.seat_id = b.seat_id WHERE b.status = 'CONFIRMED' GROUP BY r.route_id ORDER BY total_bookings DESC LIMIT 5;
